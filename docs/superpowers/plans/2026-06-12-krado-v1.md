# Krado v1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build Krado v1 — WhatsApp-first, MoMo-native booking + daily-money platform for Ghanaian barbers — per `docs/PRD.md`, `docs/ARCHITECTURE.md`, `docs/DESIGN_SYSTEM.md`.

**Architecture:** One Cloudflare Worker (Hono + TypeScript) serves the REST API, Paystack/WhatsApp webhooks, cron jobs, queue consumer, SSR public booking pages, and the React PWA via static assets. D1 is source of truth; KV holds slots/sessions; R2 stores portfolio photos. Shared domain logic (money, phones, time, booking state machine, zod schemas, i18n) lives in `packages/shared`; design-system components in `packages/ui`.

**Tech Stack:** Cloudflare Workers, Hono, D1, KV, R2, Queues, Cron Triggers, Paystack (mobile_money), WhatsApp Business Cloud API, React 18 + Vite PWA, zod, vitest (+ @cloudflare/vitest-pool-workers for the api app).

**Decisions locked from PRD open questions:** deposit default 25% (floor GHS 5); susu tile on dashboard; handles first-come (`krado.app/kojo`).

### Task 9: Paystack init + webhook (truth) + refunds

**Files:**
- Create: `apps/api/src/lib/paystack.ts` (initialize w/ `channels:["mobile_money"]`, amounts in pesewas; `verifySignature(body, sig, secret)` HMAC-SHA512 via WebCrypto; `verifyTransaction(ref)`), `apps/api/src/routes/webhooks-paystack.ts`, `apps/api/src/routes/booking-pay.ts` (`POST /api/bookings/:holdToken/pay`)
- Test: `apps/api/test/paystack.test.ts`

Webhook flow (the invariant set):
```
verify x-paystack-signature (HMAC-SHA512 of raw body) → 401 on mismatch
INSERT INTO webhook_events(provider,event_id) → conflict = already processed, 200 exit
re-verify via GET /transaction/verify/{reference} (mocked in tests)
hold still in KV? → create client (phone-first upsert) + booking (status 'locked',
  denormalized service_name/price/duration) + payments row, delete hold,
  enqueue booking_confirmed_client + booking_confirmed_artisan
hold expired? → payments row with status, flag for manual recon (no booking)
```
Tests: signature mismatch → 401 + no writes; duplicate event_id → single booking; expired-hold path; artisan-cancel → refund call + `cancelled_by_artisan` + `refund_notice` enqueued.
- [ ] TDD loop; commit `feat(api): Paystack MoMo deposits, webhook-first locking, refunds`

**Files:**
- Create: `apps/api/src/cron.ts` dispatching on `event.cron`:
